🎯 KEY PATTERNS

┌─ Component Expansion Pattern ──────────────────────────────────────────────────┐
│  ACC        → [ACC I, ACC II, ACC III, ACC IV]  (Expande automaticamente)    │
│  TCC        → [TCC I, TCC II]                    (Expande automaticamente)    │
│  Específico → [Específico]                       (Processa direto)           │
└────────────────────────────────────────────────────────────────────────────────┘

┌─ Dynamic Module Import ────────────────────────────────────────────────────────┐
│  if component.startswith("TCC"):                                              │
│      import matricular_tcc                                                    │
│  else:                                                                         │
│      import matricular                                                        │
│                                                                                │
│  Cada componente tem seu próprio fluxo Playwright                             │
└────────────────────────────────────────────────────────────────────────────────┘

┌─ Repository Pattern ───────────────────────────────────────────────────────────┐
│  Domain          →   Interface (ILancamento, IAluno, etc)                     │
│  Infrastructure  →   Implementation (LancamentoRepository, etc)               │
│  Desacoplamento perfeito entre lógica e persistência                          │
└────────────────────────────────────────────────────────────────────────────────┘

┌─ React Query Pattern ──────────────────────────────────────────────────────────┐
│  useQuery(['lancamentos', tipo])                                              │
│      └─ Fetch + Cache + Auto-refetch                                         │
│                                                                                │
│  useMutation({                                                                │
│      mutationFn: () => POST /endpoint,                                       │
│      onSuccess: () => invalidateQueries(['lancamentos'])                     │
│  })                                                                            │
│      └─ Mutation + Cache invalidation + Auto-refetch                         │
└────────────────────────────────────────────────────────────────────────────────┘

📊 MAIN FLOWS

┌─ Flow 1: Matriculation & Consolidation (Matrícula) ──────────────────────────┐
│  Secretary clicks Matricular
│          ↓
│  Frontend POST /api/admin/lancamentos/matricular
│          ↓
│  LancamentoService.matricular()
│          ├─ Expand: ACC → [ACC I, II, III, IV]
│          ├─ For each component:
│          │  ├─ Import correct module (matricular.py or matricular_tcc.py)
│          │  ├─ Run Playwright automation
│          │  └─ Collect success/error
│          ├─ Update database for successful components
│          └─ Return ResultadoOperacao
│          ↓
│  Frontend invalidates cache
│          ↓
│  Table re-renders with updated status
│          ↓
│  Secretary sees ✓ (Sim) or ✗ (Não)
└────────────────────────────────────────────────────────────────────────────────┘

┌─ Flow 2: RAG Chat (Diretor Virtual) ───────────────────────────────────────────┐
│  User types question in chat
│          ↓
│  Generate embedding of question
│          ↓
│  Vector Store finds similar docs (k=5)
│          ↓
│  LangChain mounts context
│          ↓
│  Claude API generates response with citations
│          ↓
│  Frontend displays response + sources
└────────────────────────────────────────────────────────────────────────────────┘

┌─ Flow 3: Form Processing ──────────────────────────────────────────────────────┐
│  User fills form (ACC/TCC/Estágio)
│          ↓
│  Frontend validates → POST /api/v1/forms/acc
│          ↓
│  Pydantic validates schema
│          ↓
│  ProcessarACC UseCase executes
│          ├─ Business logic
│          ├─ Save to database
│          └─ Send email notification
│          ↓
│  Frontend shows confirmation
└────────────────────────────────────────────────────────────────────────────────┘
